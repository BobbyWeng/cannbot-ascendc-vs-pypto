# Local Artifacts (Not Tracked in Git)

## Large Local-Only Files

The following files are excluded from formal archives and Git tracking:

### Raw Profiler Timeline Data
- Located in `operators/{op}/reports/raw/` per operator
- Contains msprof `PROF_*` directories with task_time_*.csv, op_summary_*.csv, SQLite databases
- Can be regenerated by running `bash benchmark/run_all.sh` on Ascend 910B hardware

### Build Output Binary Files
- `operators/{op}/ascendc/build/{op}_ascendc` — compiled kernel binary
- `operators/{op}/ascendc/build/output/output_b*.bin` — correctness output
- Can be regenerated by: `cmake -S ascendc -B build && cmake --build build -j`

### Data Binary Files
- `operators/{op}/data/*.bin` — input/reference binary files
- Can be regenerated by: `python3 data/generation_scripts/generate_inputs.py`

### Ascend C Build Artifacts
- `CMakeCache.txt`, `*.o`, `CMakeFiles/`, `Makefile`, `autogen/`
- These are build temporaries and should NOT be tracked

### JIT Cache
- PyPTO JIT compilation outputs (framework-managed)

### Archive Policy
Per AGENTS.md archive policy:
- **Include**: Source, configs, correctness results, parsed profiler, final reports, manifests, SHA256SUMS
- **Exclude**: Build cache, raw profiler, JIT cache, __pycache__, large raw data (>20MB)

Current archives (v4) in `archives/` follow this policy correctly.
